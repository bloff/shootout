    5 require tasker.fs
    6 
    7 \ read NUM from last command line argument
    8 0. argc @ 1- arg >number 2drop drop constant NUM
    9 
   10 Variable pcount
   11 Variable data
   12 Variable produced
   13 Variable consumed
   14 
   15 \ note: no mutex is needed here. bigFORTH's tasker is cooperative
   16 \ and switches tasks only with PAUSE.
   17 
   18 : producer ( n -- )
   19   next-task swap 2 $1000 NewTask pass
   20   0 ?DO
   21      BEGIN  pcount @ 1 =  WHILE  pause  REPEAT
   22      1 pcount ! I data !
   23      1 produced +!
   24   LOOP wake ;
   25 
   26 : consumer ( n -- )
   27   next-task swap 2 $1000 NewTask pass
   28   0 swap 0 ?DO
   29      BEGIN  pcount @ 0=  WHILE  pause  REPEAT
   30      0 pcount ! drop data @
   31      1 consumed +!
   32   LOOP drop wake ;
   33 
   34 NUM producer
   35 NUM consumer
   36 
   37 \ There is no "main" task - to synchronize, each of the two new
   38 \ threads get the task address of the starting task, and wake it
   39 \ when they are done. The main task therefore has to stop twice
   40 \ (and wait to be woken up)
   41 
   42 stop stop
   43 
   44 produced @ .
   45 consumed @ 1 u.r cr
   46 
   47 bye \ th-th-that's all folks!
